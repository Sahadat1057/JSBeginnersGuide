Concept: Event Loop

Explanation:

The event loop is responsible for executing and processing code. In it's default form the event loop waits for a 'message' to be recieved, proccesses the 'message' and runs the 'message.' The event loop also allows for asynchronous programming where two different functions can be ran at the same time. A common example of this is when one sets a timer of some sort using the setTimeOut() function.

In this example, begin is logged onto the console to show where the program starts. Then the asynchronous function setTimeout is used which will log 'This will run after 2 seconds' after 2 seconds have passed since running the program. As a result of setTimeout being asynchronous the contents of setTimeout() will run after 'end' is logged.
